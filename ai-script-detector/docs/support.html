Support channel

Use GitHub Issues for bug reports and support requests.

ScriptLens uses GitHub Issues as the public support route. Include the extension version, Chrome version, and the YouTube video URL when you report a bug.

Before filing an issue

  • Reload the extension from chrome://extensions and refresh the YouTube tab
  • Confirm you are on a desktop YouTube watch page, not Shorts or m.youtube.com
  • Try the inline button first, then use the workspace if you need more detail
  • If the issue involves transcript recovery, confirm whether the optional local helper is enabled

Common questions

Why does the extension say the transcript is unavailable?

Some videos expose limited transcript data or block transcript text on the page. ScriptLens treats that honestly. You can allow a title and description fallback, or optionally use the local helper for harder transcript recovery cases.

Does ScriptLens work outside YouTube?

Not in this Chrome Web Store release. The extension is intentionally scoped to desktop youtube.com/watch pages.

What should I include in a support report?

  • The exact YouTube URL
  • Whether the issue appears in the inline widget, popup, or side panel
  • Whether fallback text or the optional local helper was enabled
  • Any visible error text from ScriptLens

Optional local helper

The helper is optional and is not part of the store package. If you want transcript recovery help for blocked videos, see the backend helper notes in the repository and make sure yt-dlp is configured before you start the helper.
